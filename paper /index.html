Content Refresh Opportunity Scoring Using Search Performance Signals

Machine Learning Capstone
FlyRank ML Internship

Abstract

This study investigates how search performance signals can be used to identify content that should be prioritized for manual review and refresh. A rule-based baseline scoring approach was developed using content freshness, impressions, and click-through rate, followed by a Random Forest model evaluated with grouped validation to reduce data leakage. The resulting recommendations are presented as a ranked action queue intended to support editorial decision-making. The findings are observational and designed for decision-support rather than automated publishing.

Introduction

Content performance naturally changes over time. Older pages may lose visibility, while refreshed content can regain search performance. This project explores how machine learning can help prioritize content review using historical search performance signals. The objective is to provide a practical workflow that supports human reviewers in identifying high-priority content for refresh.

Dataset

The project uses the FlyRank ML Internship dataset containing anonymized search performance information. Features include content age, impressions, click-through rate (CTR), and additional search metrics. No client-identifying information or private search queries are included in this public report.

Methodology

  • Baseline rule-based scoring
  • Random Forest classification model
  • Grouped validation using client identifiers
  • Feature leakage audit
  • Content action playbook generation

Results

This section presents the comparison between the baseline approach and the machine learning model. Figures generated during the notebooks will be added here, including action distribution and model evaluation charts.

Recommendations

  • Refresh high-priority content identified by the model.
  • Review medium-priority content before publication decisions.
  • Continue monitoring healthy content.
  • Require human review before implementing any recommendation.

Limitations

The recommendations are based on historical observations from the available dataset. They should be interpreted as decision-support rather than proof of causation. Editorial review remains essential before any content changes are made.
